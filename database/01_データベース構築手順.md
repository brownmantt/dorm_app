## 2. データベースとロールの作成

管理ユーザー（例：`postgres`）で接続し、専用ロールとデータベースを作成します。

```sql
-- アプリ用ロール（パスワードは環境に合わせて変更）
CREATE ROLE dorm_app WITH LOGIN PASSWORD '0708';

-- パスワード変更の場合
ALTER ROLE dorm_app WITH PASSWORD '0708';

-- データベース（所有者をアプリ用ロールに）
CREATE DATABASE dormitory
    OWNER dorm_app
    ENCODING 'UTF8'
    LC_COLLATE 'ja_JP.UTF-8'
    LC_CTYPE 'ja_JP.UTF-8'
    TEMPLATE template0;
```

> Windows 環境などロケールが使えない場合は `LC_COLLATE` / `LC_CTYPE` を
> 省略（`TEMPLATE template0` のみ）しても構いません。

## 3. スキーマ（DDL）の適用

`schema.sql` を対象データベースに対して実行します。

```bash
# Windows では UTF-8 クライアント encoding を指定（文字化け防止）
# PowerShell: $env:PGCLIENTENCODING = "UTF8"

# アプリ用ロールで接続して DDL を流す
psql -h localhost -U dorm_app -d dormitory -f schema.sql
```

> `schema.sql` 先頭に `SET client_encoding TO 'UTF8';` を含みます。
> Windows 環境では上記の環境変数設定も併用してください。

> `CREATE EXTENSION btree_gist` を含みます。一般に拡張作成には
> スーパーユーザー権限が必要です。権限エラーになる場合は、
> 先に管理ユーザーで以下を実行してください。
>
> ```bash
> psql -h localhost -U postgres -d dormitory -c "CREATE EXTENSION IF NOT EXISTS btree_gist;"
> ```

### 適用されるオブジェクト（概要）

| 分類 | テーブル |
|------|----------|
| 寮・部屋 | `dormitory`, `room` |
| 入退寮 | `employee`, `residence_history`, `employee_first_dorm_use` |
| 寮費 | `fee_rate_config`, `dorm_fee` |
| 備品 | `equipment`, `equipment_moveout`, `equipment_storage` |
| ログ | `operation_log` |
| マスタ | `region`, `usage_type`, `unit_price`, `affiliation` |
| Excel取込 | `excel_import_job`, `excel_import_error` |

### 主な制約

- 性別整合・列挙値は `CHECK` 制約で担保（`gender_type`, `employee_category`, `disposition`, `status` 等）
- 入居期間の妥当性：`move_out_date IS NULL OR move_out_date >= move_in_date`
- 同一部屋の**定員超過防止**：`trg_residence_room_capacity` トリガー（論理削除レコードは対象外）
- 寮費の重複登録防止：`(employee_id, target_year_month)` の部分一意インデックス（`deleted_at IS NULL`）
- 寮マスタ `dormitory` に最寄駅カラム `nearest_station_1` / `nearest_station_2` / `nearest_station_3`（各 `VARCHAR(20)`、NULL 可）を保持
- 社員マスタ `employee` に最寄駅カラム `nearest_station_1` / `nearest_station_2` / `nearest_station_3`（各 `VARCHAR(20)`、NULL 可）を保持
- 単価マスタ `unit_price`：`daily_unit_price >= 0`、`dormitory_id` / `room_id` は NULL 可
- 利用形態マスタ `usage_type`：`min_usage_days >= 1`（未指定時 1）、`max_usage_days >= -1`（-1 は制限なし）

### 既存 DB への差分適用（マイグレーション）

`schema.sql` 適用済みの DB に後からカラムを追加する場合は、以下を実行します。

```bash
psql -h localhost -U dorm_app -d dormitory -f migration/V20260607_add_dormitory_nearest_stations.sql
psql -h localhost -U dorm_app -d dormitory -f migration/V20260607_add_employee_nearest_stations.sql
psql -h localhost -U dorm_app -d dormitory -f migration/V20260627_region_master.sql
psql -h localhost -U dorm_app -d dormitory -f migration/V20260627_usage_type_master.sql
psql -h localhost -U dorm_app -d dormitory -f migration/V20260627_unit_price_master.sql
psql -h localhost -U dorm_app -d dormitory -f migration/V20260627_unit_price_optional.sql
psql -h localhost -U dorm_app -d dormitory -f migration/V20260627_unit_price_daily_zero.sql
psql -h localhost -U dorm_app -d dormitory -f migration/V20260628_residence_usage_type.sql
psql -h localhost -U dorm_app -d dormitory -f migration/V20260628_unit_price_min_usage_days.sql
psql -h localhost -U dorm_app -d dormitory -f migration/V20260629_usage_type_usage_days.sql
psql -h localhost -U dorm_app -d dormitory -f migration/V20260628_dorm_fee_restructure.sql
```

> 新規構築時は `schema.sql` に上記テーブル・制約が含まれるため、マイグレーションは **既存 DB への差分適用時のみ** 実行してください。

## 4. 初期設定データの投入（任意・推奨）

寮費算定（F-06）には `room_type` 別の単価が必要です。`seed_config.sql` で
暫定単価を投入します（業務データは含みません）。

```bash
psql -h localhost -U dorm_app -d dormitory -f seed_config.sql
```

> 単価は研修用の暫定値です。実運用では運用設計に合わせて更新してください。

> `seed_data.sql` にも `fee_rate_config` の追加行（改定単価）が含まれます。
> 第 4 章を省略した場合でも、`seed_data.sql` 単独実行で単価の最低限投入は可能です
> （`ON CONFLICT DO NOTHING` により重複投入はスキップされます）。

## 5. 業務テストデータの投入（任意・検証環境のみ）

画面結合テスト・API 動作確認用に、寮・部屋・社員・入居履歴などのサンプルデータを
一括投入します。**本番 DB には適用しないでください。**

### 投入内容（概要）

| 分類 | テーブル | 件数 | 備考 |
|------|----------|------|------|
| 寮・部屋 | `dormitory`, `room` | 各 5 件 | 男子寮・女子寮、部屋種別 STANDARD/SMALL/OTHER |
| 入退寮 | `employee`, `residence_history` | 各 5 件 | 性別整合・期間非重複を満たす |
| 入退寮 | `employee_first_dorm_use` | **3 件** | 日本社員（`JAPAN`）のみ（設計上の制約） |
| 寮費 | `fee_rate_config`, `dorm_fee` | 各 5 件 | 仮定・エラーステータス混在 |
| 備品 | `equipment`, `equipment_moveout`, `equipment_storage` | 各 5 件 | 退去済み入居履歴に紐づけ |
| ログ | `operation_log` | 5 件 | 主要 `operation_type` を網羅 |
| Excel取込 | `excel_import_job`, `excel_import_error` | 各 5 件 | 成功・失敗ジョブ混在 |

### 実行手順

第 3 章（DDL）完了後、第 4 章（単価設定）の後または省略後に実行します。

```bash
# PowerShell（Windows）
$env:PGCLIENTENCODING = "UTF8"
psql -h localhost -U dorm_app -d dormitory -f seed_data.sql
```

> 再投入する場合は第 9 章の手順で DB を作り直すか、
> 主キー衝突を避けるため既存データを削除してから実行してください。

## 8. バックアップ・運用メモ

```bash

# リストア
```

## 10. トラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| `CREATE EXTENSION` で権限エラー | 一般ロールに拡張作成権限がない | スーパーユーザーで `btree_gist` を先に作成 |
| EXCLUDE 制約作成エラー | `btree_gist` 未導入 | 拡張を作成後に `schema.sql` を再実行 |
| ロケールエラー | OS にロケール未導入 | `LC_COLLATE/LC_CTYPE` を省略して DB 作成 |
| 文字化け | クライアント encoding 不一致 | `PGCLIENTENCODING=UTF8` を設定するか `SET client_encoding TO 'UTF8';` |
| `seed_data.sql` で主キー重複 | 同一 DB へ再実行 | 第 9 章で DB 再作成するか、既存データを削除してから再実行 |
