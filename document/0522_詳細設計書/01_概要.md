# 1. 概要

> 親文書: [README.md](./README.md)

## 1.1 目的

本書は、要件定義書およびシステム構造図に基づき、寮管理システムの実装に必要な**機能・画面・API・データ・業務ロジック**を詳細化する。開発・単体テスト・結合テストの基準文書とする。

## 1.2 設計スコープ

| 区分 | 対象 | 備考 |
|------|------|------|
| **中心機能** | **寮割カレンダー（F-14）** | RFP A版。Excel 寮割表の Web 再現 |
| 対象 | 寮・部屋、入退寮、責任者・所属、寮費、CSV/Excel 取込、印刷、非機能 | RFP `寮管理システム 要件定義書（詳細版)A.docx` 準拠 |
| 補助機能（前版継続） | 備品、空き室、初回利用日・長期利用、操作ログ | 画面は残置。優先度 Should |
| 今回対象外 | 入居時備品準備（F-13） | マスタ・退去処理・保管は対象 |
| 拡張（DB のみ先行） | 日用品費、連絡先・現場最寄駅等の表示列 | RFP 6.1。UI 表示は段階導入 |

## 1.3 前提・制約

| 区分 | 内容 |
|------|------|
| 規模 | 寮約15戸、3DK中心、1戸最大3名、同時利用約20名 |
| 所在地 | 東京・大阪等（住所はマスタ管理） |
| 入居者 | 日本社員（正社員・契約）、中国出張社員 |
| 技術スタック（暫定） | フロント：React 18 + TypeScript / API：Spring Boot 3 + Java 17 / DB：PostgreSQL 15 |
| 認証 | 社内 IdP または簡易ロール認証（本番化時に確定） |
| 削除方針 | 全マスタ・履歴は**論理削除**（`deleted_at`） |
| 履歴保持 | 5年以上（アーカイブ方針は運用設計で補足） |

## 1.4 用語定義

| 用語 | 定義 |
|------|------|
| 寮割 | 月単位で寮・入居者・在籍日を一覧化した業務帳票。本システムでは SC-01 カレンダー画面で再現 |
| 寮（戸） | 1物件単位。地域（東京/大阪/名古屋/その他）で分類 |
| 部屋 | 寮内の個室。原則 **1部屋1名**（日単位で重複不可） |
| 入居履歴 | 社員×部屋×期間（入寮日〜退寮日）の記録。業務の中心テーブル |
| 現在の部屋への入寮日 | 当該部屋への入居開始日。`residence_history.move_in_date` に対応 |
| 退寮日 NULL | 無期限入居。内部比較では `9999-12-31` 相当 |
| 責任者 | 寮ごとに最大1名。カレンダー上 ★ 表示（任意設定） |
| 所属 | 入居者の所属区分。`affiliation` マスタで管理（固定値ではない） |
| 初回利用日 | （補助）日本社員が初めて寮を使用した入寮日。部屋移動後も不変 |
| 空き室 | （補助）現在有効な入居がなく将来入居予定もない部屋 |
