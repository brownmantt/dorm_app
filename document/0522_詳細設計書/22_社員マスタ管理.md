# 22 社員マスタ管理（F-21 / SC-16）

## 1. 概要

| 項目 | 内容 |
|------|------|
| 機能 ID | F-21 |
| 画面 ID | SC-16 |
| 画面名 | 社員マスタ |
| 権限 | 管理者（ROLE_ADMIN） |
| ルート | `/employees` |

人事システム連携前の暫定手段として、社員（入居者）の基本情報を手登録・更新・論理削除する。
入居登録（SC-06）・寮割カレンダー（SC-01）の社員検索選択のデータソースとなる。

---

## 2. 画面項目

### 2.1 検索条件

| 項目 | 入力 | 備考 |
|------|------|------|
| キーワード | テキスト | 社員ID / 氏名の部分一致 |
| 性別 | 選択 | MALE / FEMALE |
| 入居者区分 | 選択 | JAPAN / CHINA_ASSIGN |
| 所属 | 選択 | 所属マスタから選択 |

### 2.2 一覧表示

| 項目 | データソース |
|------|-------------|
| 社員ID | employee.employee_id |
| 氏名 | employee.name |
| 性別 | employee.gender |
| 入居者区分 | employee.employee_category |
| 所属 | affiliation.name（LEFT JOIN） |
| 事業部 | employee.business_division |
| 最寄駅 | employee.nearest_station |
| 連絡先 | employee.contact_info（mobilePhone / email） |

### 2.3 登録・編集ダイアログ

| 項目 | 入力 | 必須 | 備考 |
|------|------|------|------|
| 社員ID | テキスト | ○（新規のみ） | 20文字以内、一意。編集時は変更不可 |
| 氏名 | テキスト | ○ | 100文字以内 |
| 性別 | 選択 | ○ | MALE / FEMALE |
| 入居者区分 | 選択 | ○ | JAPAN / CHINA_ASSIGN |
| 所属 | 選択 | — | affiliation_id |
| 事業部 | テキスト | — | 30文字以内 |
| 最寄駅 | テキスト | — | 100文字以内 |
| 携帯電話 | テキスト | — | contact_info.mobilePhone |
| メール | テキスト | — | contact_info.email |

## 3. API

| メソッド | URL | 概要 | 権限 |
|---------|-----|------|------|
| GET | `/api/v1/employees` | 社員一覧（検索・ページング） | ADMIN / USER |
| GET | `/api/v1/employees/{empId}` | 社員詳細 | ADMIN / USER |
| POST | `/api/v1/employees` | 社員新規登録 | ADMIN |
| PUT | `/api/v1/employees/{empId}` | 社員更新 | ADMIN |
| DELETE | `/api/v1/employees/{empId}` | 社員論理削除 | ADMIN |

既存 API（変更なし）:

| GET | `/api/v1/employees/{empId}/first-use-date` | 初回利用日 |
| GET | `/api/v1/employees/{empId}/total-usage-days` | 通算利用日数 |

## 4. 業務ルール

1. 社員IDは未削除レコード内で一意（`EMPLOYEE_ID_DUPLICATE`）。
2. 所属IDを指定する場合、所属マスタに存在すること（`AFFILIATION_NOT_FOUND`）。
3. 入居履歴（`residence_history`）が1件以上ある社員は削除不可（`EMPLOYEE_IN_USE`）。
4. 削除は論理削除（`deleted_at` 設定）。物理削除は行わない。
5. 操作ログに CREATE / UPDATE / DELETE を記録する。

## 5. 呼び出しチェーン

```
views/employee/EmployeeList.vue  fetchList() / handleSubmit()
 ↓
api/employee.js  getEmployees() / createEmployee() / updateEmployee() / deleteEmployee()
 ↓
EmployeeController → EmployeeService → EmployeeMapper → employee / affiliation
```

入居登録との連携:

```
views/residence/ResidenceRegister.vue  searchEmployeeOptions()
 ↓
api/employee.js  searchEmployees({ keyword })
 ↓
GET /employees（SC-16 と同一 API）
```

## 6. エラーコード

| コード | HTTP | 説明 |
|--------|------|------|
| EMPLOYEE_NOT_FOUND | 404 | 社員が見つかりません |
| EMPLOYEE_ID_REQUIRED | 400 | 社員IDを入力してください |
| EMPLOYEE_ID_DUPLICATE | 400 | 社員IDが重複しています |
| EMPLOYEE_IN_USE | 400 | 入居履歴が存在する社員は削除できません |
| AFFILIATION_NOT_FOUND | 404 | 所属が見つかりません |
| INVALID_GENDER | 400 | 性別が不正です |
| INVALID_EMPLOYEE_CATEGORY | 400 | 入居者区分が不正です |
