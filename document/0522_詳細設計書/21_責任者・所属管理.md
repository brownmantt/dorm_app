## 画面設計

| 画面 ID | 画面名 | 権限 | 備考 |
|---------|--------|------|------|
| SC-15 | 所属マスタ | 管理者 | F-20 |
| SC-03 | 寮詳細 | 一般/管理者 | F-15 責任者設定を追加 |

### SC-03 寮詳細 — 責任者設定（F-15）

| 項目 | 入力 | 必須 | 備考 |
|------|------|------|------|
| 責任者 | 入居者から選択 | 任意 | 当該寮に**現在有効な入居者**のみ選択可 |
| 表示 | ★マーク | — | SC-01 寮割カレンダーで表示 |

**ルール（RFP 4.2）**

| ルール | 内容 |
|--------|------|
| 人数 | 寮ごとに最大 **1 名** |
| 必須 | **任意**（0 名可） |
| 対象者 | 当該寮の現在有効入居者に限定 |
| 自動設定 | 責任者未設定時、**全室の現入居者のうち最も早い入寮者**を自動登録（手動変更可） |
| 退寮時 | 責任者が退寮した場合、責任者設定を自動解除 |

### SC-15 所属マスタ

| 項目 | 入力 | 必須 | 備考 |
|------|------|------|------|
| 所属コード | テキスト | ○ | 一意 |
| 所属名称 | テキスト | ○ | 例：瀋陽、CDXT、社員 |
| 表示順 | 数値 | 任意 | 一覧ソート用 |

**無効化**: 有効フラグ項目は持たない。削除操作により論理削除（`deleted_at` 設定）する。

| DELETE | `/api/v1/dormitories/{id}/manager` | F-15 | 責任者解除 |

### PUT `/api/v1/dormitories/{id}/manager`

**Request**

```json
{
  "employeeId": "E00012",
  "residenceHistoryId": "RH10045"
}
```

**エラー**

| code | 条件 |
|------|------|
| `MANAGER_NOT_RESIDENT` | 指定社員が当該寮の現在有効入居者でない |
| `MANAGER_ALREADY_EXISTS` | 既に別の責任者が設定済み（上書き時は PUT で置換可） |

## データベース

### `affiliation`（所属マスタ）

| カラム | 型 | NULL | 説明 |
|--------|-----|------|------|
| affiliation_id | VARCHAR(20) | NO | PK |
| code | VARCHAR(30) | NO | 所属コード（一意） |
| name | VARCHAR(100) | NO | 所属名称 |
| display_order | INT | YES | 表示順 |
| created_at | TIMESTAMPTZ | NO | |
| updated_at | TIMESTAMPTZ | NO | |
| deleted_at | TIMESTAMPTZ | YES | 論理削除 |

### `dormitory_manager`（寮責任者）

| カラム | 型 | NULL | 説明 |
|--------|-----|------|------|
| dormitory_id | VARCHAR(20) | NO | PK, FK → dormitory |
| employee_id | VARCHAR(20) | NO | FK → employee |
| residence_history_id | VARCHAR(20) | NO | FK → residence_history（現入居の紐づけ） |
| assigned_at | TIMESTAMPTZ | NO | 設定日時 |
| version | INT | NO | 楽観ロック（既定 0） |

**制約**: `dormitory_id` に対して有効レコードは最大 1 件。

### `employee` 追加カラム

| カラム | 型 | NULL | 説明 |
|--------|-----|------|------|
| affiliation_id | VARCHAR(20) | YES | FK → affiliation |
| business_division | VARCHAR(30) | YES | 事業部（拡張・サンプル準拠） |
| nearest_station | VARCHAR(100) | YES | 現場最寄駅（拡張） |
| contact_info | JSONB | YES | 連絡先（拡張） |

## 業務ロジック

### `DormitoryManagerService.assign`

```text
1. 寮の存在確認
2. 指定 residence_history が当該寮・現在有効入居であることを検証
3. dormitory_manager を UPSERT（既存があれば employee 差し替え）
4. 操作ログ記録
```

### `AffiliationService`

標準 CRUD + 論理削除。削除時、参照中の `employee` が存在する場合は削除不可（`AFFILIATION_IN_USE`）。
