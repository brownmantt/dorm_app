### SC-02 寮一覧（新規登録・編集ダイアログ）

| 項目 | 入力 | 必須 | 備考 |
|------|------|------|------|
| 寮名称 | テキスト | ○ | |
| 郵便番号 | テキスト | ○ | 7桁（`123-4567` 形式表示可）。blur 時に住所自動取得 |
| 住所 | テキスト | ○ | 郵便番号から自動入力。手動修正可 |
| 地域 | 選択 | ○ | **地域マスタ**（`region.code`）から動的取得。初期値: 東京 / 大阪 / 名古屋 / その他 |
| 間取り | 選択 | ○ | 3DK / 2DK 等 |
| 種別 | 選択 | ○ | 男性寮 / 女性寮（登録後変更不可） |
| 備考 | テキスト | 任意 | |

**郵便番号→住所自動反映**：7桁入力完了後（フォーカスアウト）に `GET /postal-codes/{postalCode}/address` を呼び出し、返却された `address` を住所欄へ設定する。

**関連エラー**: `DORM_GENDER_CHANGE_FORBIDDEN`、`INVALID_POSTAL_CODE`、`POSTAL_CODE_NOT_FOUND`、`POSTAL_CODE_LOOKUP_FAILED` — [15_エラー・例外設計.md](./15_エラー・例外設計.md)

### GET `/api/v1/postal-codes/{postalCode}/address` — 郵便番号住所検索

zipcloud API 経由で都道府県・市区町村・町域を取得し、住所自動入力に利用する。

**Response 200**

```json
{
  "postalCode": "1000001",
  "prefecture": "東京都",
  "city": "千代田区",
  "town": "千代田",
  "address": "東京都千代田区千代田"
}
```

### POST `/api/v1/dormitories` — 寮登録（抜粋）

**Request**（`postalCode` 必須）

```json
{
  "name": "東京寮A",
  "postalCode": "1000001",
  "address": "東京都千代田区千代田",
  "region": "TOKYO",
  "layoutType": "3DK",
  "genderType": "MALE",
  "remarks": null
}
```

## 業務ロジック

### 7.1.1 寮登録 `DormitoryService.create`

1. `postal_code` を7桁数字に正規化し必須検証
2. `gender_type` ∈ {`MALE`, `FEMALE`} を検証
3. 同一 `name` + `address` の重複警告（任意、論理削除除外）
4. 登録後、間取りに応じた部屋数の**初期部屋は手動登録**（自動生成はしない）

### 7.1.2 郵便番号住所検索 `PostalCodeService.lookupAddress`

1. 郵便番号を7桁数字に正規化（ハイフン除去）
2. zipcloud API（`https://zipcloud.ibsnet.co.jp/api/search`）を呼び出し
3. 都道府県・市区町村・町域を結合した `address` を返却
4. 該当なし・外部 API 失敗時は業務エラーを返却

**呼び出しチェーン（フロント）**

```text
SC-02 郵便番号 blur
 → GET /postal-codes/{postalCode}/address
 → 住所欄へ自動反映
 → POST /dormitories（postalCode 含む）
```

### 7.1.3 部屋登録 `RoomService.create`

1. 親寮が存在し `deleted_at IS NULL`
2. `capacity` 既定値 1（未指定時）
3. `area_sqm` > 0

### 7.1.4 寮削除 `DormitoryService.delete`

1. 対象寮が存在し `deleted_at IS NULL`
2. 当該寮に**現在有効入居者**（`move_out_date IS NULL` または基準日時点で在籍）が1名以上いる場合は削除不可（`DORMITORY_IN_USE`）
3. 論理削除を実行

**現在有効入居の定義**（[08_入退寮管理.md](./08_入退寮管理.md) と同一）:

- `move_in_date <= 基準日`
- `move_out_date IS NULL OR move_out_date >= 基準日`

### 7.1.5 部屋削除 `DormitoryService.deleteRoom`

1. 対象部屋が存在し `deleted_at IS NULL`
2. 当該部屋に現在有効入居者が1名以上いる場合は削除不可（`ROOM_IN_USE`）
3. 論理削除を実行
