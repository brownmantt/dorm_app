| 部屋 | コンボボックス | いいえ | 寮選択時のみ選択可（任意） |

### SC-11 備品保管

| 区分 | 項目 | 入力 | 必須 | 備考 |
|------|------|------|------|------|
| 検索 | 備品 | コンボボックス | 任意 | 備品（個体）で絞込 |
| 一覧 | 備品番号 / 品目名称 / 保管場所 / 保管数量 / ステータス | 表示 | — | ページング。1備品×複数保管場所は複数行表示 |
| ツールバー | 保管登録 | ボタン | — | 明細部左上。ページネーションと同一行 |
| 登録・編集 | 備品 | コンボボックス | ○ | **備品（個体）**参照。編集時は変更不可 |
| 登録・編集 | 保管明細 | 表形式 | ○ | 保管場所・保管数量・ステータス・関連退去ID |
| 登録・編集 | 行追加 | ボタン | 条件付き | 購入数量 &gt; 1 のときのみ表示。最大行数は購入数量以下 |

**業務ルール**

| ルール | 内容 |
|--------|------|
| 参照先 | 備品は `equipment_asset`（購入数量を保持）を参照 |
| 複数保管場所 | 購入数量が **2 以上**のとき、複数の保管場所に分割登録可能 |
| 数量整合 | 同一備品の保管数量合計は **購入数量と完全一致**すること（サーバー検証） |
| 購入数量 = 1 | 保管明細は **1 行のみ**、保管数量は **1 固定** |
| 保管場所の重複 | 同一備品内で同じ保管場所を複数行に指定不可 |
| 保存方式 | 備品ごとに既存保管明細を置換（`PUT /equipment-storages/by-asset/{id}`） |
| 削除 | 行単位削除可（`DELETE /equipment-storages/{storageId}`）。備品削除時は保管明細がある場合は不可 |

## 業務ロジック

| 機能 | 処理概要 |
|------|----------|
| マスタ CRUD | 標準 CRUD + 論理削除 |
| 備品 CRUD | 品目コンボ選択・備品番号自動採番（14桁: `EB` + yyyyMMdd + 4桁連番）・購入数量・購入情報・保証期限・備考の CRUD + 論理削除 |
| 備品登録 | 購入数量の値に関わらず **1件のみ** レコード作成。購入数量はデータ項目として各レコードに保持 |
| 備品利用登録 | 寮・部屋・入居者を指定して利用レコードを作成。利用中（`usage_end_date` IS NULL）の `usage_quantity` 合計 + 新規個数が `equipment_asset.purchase_quantity` を超えないことを検証 |
| 備品利用解除 | 既存利用レコードに `usage_end_date` を設定（入居履歴・退去備品処理との紐付けなし） |
| 退去処理 | 単独画面なし（API `POST /equipment-moveouts` は残置・他用途） |
| 保管 | 備品（個体）ごとに保管場所マスタを参照して `equipment_storage` へ登録。**保管数量合計 = 購入数量**を検証。購入数量 ≥ 2 のとき複数保管場所に分割可能 |
| 入居時準備 | **実装しない**（API・画面なし） |

操作ログ: `EQUIPMENT_USAGE_CREATE` / `EQUIPMENT_USAGE_RELEASE`、`EQUIPMENT_MOVEOUT` — [13_操作ログ.md](./13_操作ログ.md)
