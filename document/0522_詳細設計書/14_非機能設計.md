# 8. 非機能設計

> 親文書: [README.md](./README.md)

| 区分 | 要件 | 設計方針 |
|------|------|----------|
| 性能 | 同時数名（RFP 11） | 寮割カレンダー API 2秒以内（95%tile）、DB コネクションプール 10 |
| 可用性 | 研修・将来本番 | 単一 AZ 構成から開始、本番化時に冗長化 |
| セキュリティ | 社内利用 | HTTPS 必須、ロールベース認可、操作ログ |
| 監査 | **3年以上**（RFP 11） | `operation_log` + `residence_change_log` + 論理削除で追跡 |
| バックアップ | 3年以上 | 日次フルバックアップ（本番化時 RPO/RTO 定義） |
| 印刷 | A4/A3 横・寮単位改ページ | ブラウザ印刷 CSS（[20_寮割カレンダー.md](./20_寮割カレンダー.md)） |
| テスト | AI評価 | Service 層ユニットテスト必須、業務制約はテーブル駆動テスト |

関連: [16_セキュリティ設計.md](./16_セキュリティ設計.md)、[13_操作ログ.md](./13_操作ログ.md)
