## 日付・カレンダー表示ロジック（F-14 核心）

### 表示対象入居者の判定

```text
対象月の月初 = monthStart
対象月の月末 = monthEnd

表示対象 =
  move_in_date <= monthEnd
  AND COALESCE(move_out_date, '9999-12-31') >= monthStart
```

**例（RFP 2.4）**

| 入寮日 | 退寮日 | 2026年4月表示 |
|--------|--------|---------------|
| 2026-03-01 | 2026-05-01 | ○（4月に1日でも在籍） |
| 2026-04-15 | NULL | ○ |
| 2026-05-01 | NULL | × |

### 日別在籍判定（黄色セル）

```text
対象日 d について在籍 =
  move_in_date <= d
  AND COALESCE(move_out_date, '9999-12-31') >= d
```

| 項目 | 仕様 |
|------|------|
| 日付型 | すべて `DATE`（時刻なし） |
| 表示範囲 | 対象月の 1 日〜月末のみ（28〜31日は自動判定） |
| 退寮日 NULL | 無期限入居。内部比較は `9999-12-31` 相当 |
| 退寮日が対象月内 | 日付列を**赤文字**、当月末日までの黄色は退寮日まで |

### 部屋重複の視覚警告（RFP 3.3）

同一部屋・同一日に複数入居者が在籍する場合（データ不整合）:

| 表示 | 内容 |
|------|------|
| 該当日セル | **赤背景** |
| ツールチップ | 「同一部屋に {n} 名の入居が重複しています（{room_name} / {date}）」 |

判定は `DormAllocationService` がカレンダー生成時に日単位で実施する。

### 退寮予定警告（F-16）

| 項目 | 内容 |
|------|------|
| 条件 | `move_out_date IS NOT NULL` かつ `move_out_date - 基準日 <= 14` |
| 表示 | 入居者行に警告アイコン、退寮日セル強調 |
| 設定値 | 初期 14 日（将来：管理画面から変更可能） |

Query は `/dorm-allocation` と同一。レスポンスに印刷レイアウト用メタデータ（`pageBreakPerDormitory: true`, `orientation: landscape`）を付与。

| 印刷仕様 | 内容 |
|----------|------|

| ページ区切り | **寮ごとに改ページ** |

| 色 | 黄色在籍・赤文字退寮日・赤背景重複を印刷 CSS で再現 |

## 業務ロジック

### `DormAllocationService.buildCalendar`

```text
1. yearMonth から月初・月末・日数・曜日を算出
2. regions / name で寮・入居履歴をフィルタ
3. 表示対象入居者を抽出（2.4 判定）
4. 寮単位にグルーピング、寮名昇順ソート
5. 各行について日別在籍配列 occupiedDays を生成
6. 同一 room_id × 日付で重複があれば conflictDays / roomConflictDays を設定
7. 退寮警告フラグを付与（F-16）
8. DormAllocationCalendarVO を返却
```

### `MoveOutWarningService`

```text
警告対象 = move_out_date IS NOT NULL
         AND move_out_date >= 基準日
         AND move_out_date - 基準日 <= warningDays（初期 14）
```
