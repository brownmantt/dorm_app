# マスタ管理（地域・利用形態・単価）（モジュール 10）

> 親文書: [README.md](./README.md)  
> 機能 ID: **F-22**, **F-23**, **F-24**  
> 要件参照: RFP 2.7（地域フィルタ）、寮費日割り算定（単価参照の基盤）  
> 主要 Service: `RegionService`, `UsageTypeService`, `UnitPriceService`

---

## 機能概要

| 機能 ID | 機能名 | 優先度 | 備考 |
|---------|--------|--------|------|
| F-22 | 地域マスタ管理 | Must | 全画面の地域コンボボックスの選択肢を提供 |
| F-23 | 利用形態マスタ管理 | Must | 入退寮の利用形態区分・**寮費算定の利用日数上下限**（min/max）を管理 |
| F-24 | 単価マスタ管理 | Must | 地域・寮・部屋・利用形態別の**日単価**を管理 |

**関連画面への影響**

- SC-01 寮割カレンダー、SC-02 寮一覧 等の地域コンボボックスは **地域マスタ（`region`）** から動的取得する（固定列挙を廃止）。
- 単価マスタは寮費算定（F-06）で参照する**日単価**のマスタデータ源とする（`fee_rate_config` とは別管理）。
- 利用形態マスタは寮費算定（F-06）で参照する**最小/最大利用日数**のマスタデータ源とする（v2.2 より単価マスタから移管）。

### SC-19 単価マスタ

#### 検索条件（1 行固定レイアウト）

| 項目 | 入力 | 必須 | 備考 |
|------|------|------|------|
| 単価コード | テキスト | 任意 | 部分一致 |
| 地域 | 選択 | 任意 | 地域マスタ参照。選択時のみ寮コンボを有効化 |
| 寮 | 選択 | 任意 | 選択地域の寮に絞込 |
| 利用形態 | 選択 | 任意 | 利用形態マスタ参照 |

#### 一覧列

単価コード、地域、寮、部屋、利用形態、日単価、操作（編集・削除）。

- 寮・部屋未設定時は `—` 表示。

#### 新規登録・編集ダイアログ

| 項目 | 入力 | 必須 | 備考 |
|------|------|------|------|
| 単価コード | 表示のみ | — | **新規時は非表示**（サーバー自動採番 `UC` プレフィックス）。編集時は読み取り専用 |
| 地域 | 選択 | ○ | 地域マスタ（`region.code`） |
| 寮 | 選択 | 任意 | 地域選択後に当該地域の寮のみ表示。クリア可 |
| 部屋 | 選択 | 任意 | 寮選択後に当該寮の部屋のみ表示。クリア可 |
| 利用形態 | 選択 | ○ | 利用形態マスタ |
| 日単価 | 数値 | ○ | 0 以上、小数第 2 位 |

**業務ルール**

| ルール | 内容 |
|--------|------|
| 単価コード採番 | 登録時 `IdGenerator.nextId("UC")` で自動採番。更新時は既存コードを維持 |
| 寮・部屋の整合 | 寮指定時は地域と寮の `region` が一致すること。部屋指定時は寮も必須で、部屋が当該寮に属すること |

## API 設計

共通仕様: [05_API共通仕様.md](./05_API共通仕様.md)  
詳細パラメータ: `document/APIdoc/region.md`, `usageType.md`, `unitPrice.md`

### 地域マスタ（F-22）

| Method | Path | 概要 |
|--------|------|------|
| GET | `/api/v1/regions` | 地域一覧（`name` 部分一致・ページング） |
| POST | `/api/v1/regions` | 地域登録 |
| PUT | `/api/v1/regions/{id}` | 地域更新 |
| DELETE | `/api/v1/regions/{id}` | 地域論理削除 |

ID プレフィックス: `RG`

### 利用形態マスタ（F-23）

| Method | Path | 概要 |
|--------|------|------|
| GET | `/api/v1/usage-types` | 利用形態一覧（`name` 部分一致・ページング） |
| POST | `/api/v1/usage-types` | 利用形態登録 |
| PUT | `/api/v1/usage-types/{id}` | 利用形態更新 |
| DELETE | `/api/v1/usage-types/{id}` | 利用形態論理削除 |

ID プレフィックス: `UT`

**登録 Request 例**

```json
{
  "code": "NORMAL",
  "name": "通常利用",
  "displayOrder": 1,
  "minUsageDays": 1,
  "maxUsageDays": null
}
```

**主な業務エラー**

| code | 条件 |
|------|------|
| `USAGE_TYPE_CODE_DUPLICATE` | コード値重複 |
| `USAGE_TYPE_MAX_DAYS_INVALID` | 最大利用日数が 1 未満（-1 以外） |
| `USAGE_TYPE_MIN_DAYS_INVALID` | 最小利用日数が 1 未満 |
| `USAGE_TYPE_MIN_MAX_DAYS_INVALID` | 最小利用日数が最大利用日数を超過 |

### 単価マスタ（F-24）

| Method | Path | 概要 |
|--------|------|------|
| GET | `/api/v1/unit-prices` | 単価一覧（コード・地域・寮・利用形態・ページング） |
| POST | `/api/v1/unit-prices` | 単価登録 |
| PUT | `/api/v1/unit-prices/{id}` | 単価更新 |
| DELETE | `/api/v1/unit-prices/{id}` | 単価論理削除 |

ID プレフィックス: `UP`（単価ID）、`UC`（単価コード・自動採番）

**登録 Request 例**

```json
{
  "region": "TOKYO",
  "dormitoryId": null,
  "roomId": null,
  "usageTypeCode": "NORMAL",
  "dailyUnitPrice": 3500.00
}
```

**主な業務エラー**

| code | 条件 |
|------|------|
| `REGION_NOT_FOUND` | 地域コードがマスタに存在しない |
| `USAGE_TYPE_NOT_FOUND` | 利用形態コードがマスタに存在しない |
| `DORMITORY_NOT_FOUND` | 寮IDが存在しない |
| `ROOM_NOT_FOUND` | 部屋IDが存在しない |
| `UNIT_PRICE_DORMITORY_REGION_MISMATCH` | 寮の地域と選択地域が不一致 |
| `UNIT_PRICE_ROOM_DORMITORY_MISMATCH` | 部屋が指定寮に属さない |
| `UNIT_PRICE_ROOM_REQUIRES_DORMITORY` | 部屋のみ指定（寮未指定） |

## データベース

### `region`（地域マスタ）

| カラム | 型 | NULL | 説明 |
|--------|-----|------|------|
| region_id | VARCHAR(20) | NO | PK（`RG` プレフィックス） |
| code | VARCHAR(30) | NO | 地域コード（一意） |
| name | VARCHAR(100) | NO | 地域名称 |
| display_order | INT | YES | 表示順 |
| created_at | TIMESTAMPTZ | NO | |
| updated_at | TIMESTAMPTZ | NO | |
| deleted_at | TIMESTAMPTZ | YES | 論理削除 |

**インデックス**: `uk_region_code`（`code`、部分一意 `deleted_at IS NULL`）

**参照**: `dormitory.region` は `region.code` を格納（FK 制約は論理参照）。`unit_price.region` も同様。

### `usage_type`（利用形態マスタ）

| カラム | 型 | NULL | 説明 |
|--------|-----|------|------|
| usage_type_id | VARCHAR(20) | NO | PK（`UT` プレフィックス） |
| code | VARCHAR(30) | NO | コード値（一意） |
| name | VARCHAR(100) | NO | 名称 |
| display_order | INT | YES | 表示順 |
| min_usage_days | INT | NO | 最小利用日数（1 以上。未指定時 1） |
| max_usage_days | INT | NO | 最大利用日数（-1＝制限なし） |
| created_at | TIMESTAMPTZ | NO | |
| updated_at | TIMESTAMPTZ | NO | |
| deleted_at | TIMESTAMPTZ | YES | 論理削除 |

**インデックス**: `uk_usage_type_code`

**制約**

- `ck_usage_type_min_days`: `min_usage_days >= 1`
- `ck_usage_type_max_days`: `max_usage_days >= -1`

### `unit_price`（単価マスタ）

| カラム | 型 | NULL | 説明 |
|--------|-----|------|------|
| unit_price_id | VARCHAR(20) | NO | PK（`UP` プレフィックス） |
| code | VARCHAR(30) | NO | 単価コード（`UC` 自動採番・一意） |
| region | VARCHAR(30) | NO | 地域コード → `region.code` |
| dormitory_id | VARCHAR(20) | YES | 寮ID（任意） |
| room_id | VARCHAR(20) | YES | 部屋ID（任意） |
| usage_type_code | VARCHAR(30) | NO | 利用形態コード → `usage_type.code` |
| daily_unit_price | DECIMAL(10,2) | NO | 日単価（0 以上） |
| created_at | TIMESTAMPTZ | NO | |
| updated_at | TIMESTAMPTZ | NO | |
| deleted_at | TIMESTAMPTZ | YES | 論理削除 |

**制約**

- `ck_unit_price_daily`: `daily_unit_price >= 0`
- `uk_unit_price_code`: `code` 部分一意

## クラス設計

```text
RegionService / RegionServiceImpl
+ list(name, page, size): PageResult<Region>
+ create(RegionSaveDTO): Region
+ update(regionId, RegionSaveDTO): Region
+ delete(regionId): void

UsageTypeService / UsageTypeServiceImpl
+ list(name, page, size): PageResult<UsageType>
+ create(UsageTypeSaveDTO): UsageType
+ update(usageTypeId, UsageTypeSaveDTO): UsageType
+ delete(usageTypeId): void
- resolveMinUsageDays(Integer): int  // null → 1
- resolveMaxUsageDays(Integer): int  // null → -1（制限なし）
- assertMinMaxDaysValid(min, max)

UnitPriceService / UnitPriceServiceImpl
+ list(code, region, dormitoryId, usageTypeCode, page, size): PageResult<UnitPriceListView>
+ create(UnitPriceSaveDTO): UnitPrice
+ update(unitPriceId, UnitPriceSaveDTO): UnitPrice
+ delete(unitPriceId): void
- validateReferences(dto)
```

**呼び出しチェーン（利用形態マスタ）**

```text

 ↓

 ↓

```

UnitPriceController → UnitPriceServiceImpl → UnitPriceMapper
