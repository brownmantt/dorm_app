# 08-test-agent フォーマットリファレンス

## Excel 列定義

| 列 | 列名 | 幅 | 説明 |
|----|------|-----|------|
| A | 所属モジュール | 18 | PRD のモジュール名 |
| B | 関連要求 | 25 | PRD 章番号またはユーザーストーリー番号 |
| C | テストケース名 | 40 | `[条件]_[動詞＋目的語]` |
| D | 前提条件 | 35 | 数字＋改行 |
| E | テストタイプ | 14 | 機能/性能/安全/互換性/インターフェース/堅牢性 |
| F | テスト手順 | 60 | 数字＋`[操作タイプ]` シーケンス |
| G | 期待結果 | 50 | 数字＋期待結果（手順と1対1） |
| H | 優先度 | 8 | P1/P2/P3/P4 |

## 操作タイプ一覧（日本語統一版）

| 操作タイプ | 書式 | 例 |
|-----------|------|-----|
| ページ遷移 | `[遷移] パス` | `1. [遷移] /login` |
| テキスト入力 | `[入力] セレクタ::値` | `2. [入力] input[name="user"]::admin` |
| クリック | `[クリック] ボタン名/セレクタ` | `3. [クリック] ログイン` |
| 待機 | `[待機] ミリ秒` | `4. [待機] 2000` |
| 可視性確認 | `[確認] テキスト/セレクタ 可視` | `5. [確認] ダッシュボード 可視` |
| URL確認 | `[URL確認] パス断片` | `6. [URL確認] /dashboard` |
| DB検証 | `[DB] SQL文` | `7. [DB] SELECT * FROM sys_user` |
| 選択 | `[選択] セレクタ::選択肢` | `8. [選択] .el-select::第一倉庫` |
| スクリーンショット | `[撮影] 説明` | `9. [撮影] ログイン成功画面` |

## 前提条件テンプレート

```
1. フロントエンド（{フロントエンドURL}）およびバックエンド（{バックエンドURL}）が起動している
2. データベース {DB名} に正常接続できる
3. {テストデータ条件}
```

## テスト手順完全例

```
1. [遷移] /login
2. [入力] input[placeholder*="ユーザー名"]::admin
3. [入力] input[placeholder*="パスワード"]::123456
4. [クリック] ログイン
5. [待機] 2000
6. [URL確認] /dashboard
7. [確認] ダッシュボード 可視
8. [DB] SELECT id, username, real_name, status FROM sys_user WHERE username='admin'
9. [撮影] ログイン成功-ダッシュボード画面
```

## 期待結果完全例

```
1. ログインページが正常に表示される
2. ユーザー名欄に "admin" が入力されている
3. パスワード欄がマスク表示される
4. ログインリクエストが送信される
5. ページが遷移する
6. URL に /dashboard が含まれる
7. 「ダッシュボード」が表示される
8. DB に admin ユーザーのレコードが存在する
9. スクリーンショットが保存される
```

## テストケース命名例

| 条件 | 例 |
|------|-----|
| 正常 | `正常ログイン_ホーム画面へ遷移することを確認` |
| 異常 | `異常ログイン_誤ったパスワードでエラー表示を確認` |
| 境界 | `境界入力_空フォームでバリデーション表示を確認` |
| 権限 | `権限チェック_権限なしユーザーが拒否されることを確認` |
| DB検証 | `DB検証_adminユーザーレコードが存在することを確認` |
